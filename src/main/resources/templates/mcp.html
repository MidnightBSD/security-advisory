MCP API

A set of tool-shaped HTTP endpoints designed for AI coding agents (via the Model Context Protocol) to query CVEs, resolve CPE identifiers, and check whether the packages installed on an operating system are affected by known vulnerabilities. Large result sets and whole-OS scans can be streamed with Server-Sent Events (SSE).

Base URL

All endpoints live under /api/mcp. On this deployment the base URL is https://sec.midnightbsd.org. Every response is JSON unless noted as an SSE stream. The full machine-readable tool manifest is available at /api/mcp/tools — point your agent there first to discover every tool and its parameters.

Endpoints

ToolMethod & PathDescription
list_tools GET /api/mcp/tools Self-describing manifest of every tool and its parameters.
get_cve GET /api/mcp/cve/{cveId} Full detail for a single CVE (e.g. CVE-2024-3094). 404 if not tracked.
search_advisories GET /api/mcp/search?term=&page=&size= Paged free-text search over CVE id and description.
stream_search GET /api/mcp/search/stream?term=&size=&maxPages= SSE stream of every search result.
match_cpe GET /api/mcp/cpe?cpe=&includeVersion=&startDate= Match advisories from a full or partial CPE 2.3 identifier.
product_advisories GET /api/mcp/product/{name} All advisories that reference a product name.
check_package GET /api/mcp/product/{name}/version/{version}?vendor= Version-aware verdict on whether one product version is vulnerable.
check_packages POST /api/mcp/check Batch-check a JSON array of {name, version, vendor?} packages.
stream_check_packages POST /api/mcp/check/stream SSE stream of per-package findings for an OS manifest; progress is persisted.
scan_status GET /api/mcp/scan/{id} Poll the status/progress of a streamed package scan.

Examples

https://sec.midnightbsd.org

# Look up a single CVE
curl https://sec.midnightbsd.org/api/mcp/cve/CVE-2024-3094

# Is sendmail 5.58 vulnerable?
curl "https://sec.midnightbsd.org/api/mcp/product/sendmail/version/5.58"

# Match a (partial) CPE
curl "https://sec.midnightbsd.org/api/mcp/cpe?cpe=cpe:2.3:a:apache:mod_dav_svn:1.14.2&includeVersion=true"

# Batch-check a list of installed packages
curl -X POST https://sec.midnightbsd.org/api/mcp/check \
  -H 'Content-Type: application/json' \
  -d '[{"name":"sendmail","version":"5.58"},{"name":"curl","version":"8.5.0"}]'

Streaming (Server-Sent Events)

The two /stream endpoints return text/event-stream. Consume them with any SSE client (or curl -N). Named events are emitted:

  • session — (scan only) first event, carries the sessionId and the total item count.
  • result — one per search hit or per package result.
  • complete — final summary; the stream then closes.
# Stream a whole-OS package scan and watch findings arrive
curl -N -X POST https://sec.midnightbsd.org/api/mcp/check/stream \
  -H 'Content-Type: application/json' \
  -d '[{"name":"openssl","version":"3.0.2"},{"name":"bash","version":"5.1"}]'

# event: session
# data: {"sessionId":"4f3c...","total":2}
# event: result
# data: {"name":"openssl","version":"3.0.2","vulnerable":true,"advisoryCount":3,"advisories":[...]}
# ...
# event: complete
# data: {"sessionId":"4f3c...","processed":2,"vulnerable":1}

# Poll progress of a long scan from another shell
curl https://sec.midnightbsd.org/api/mcp/scan/4f3c...

Using with AI coding CLIs

These are plain HTTP + SSE endpoints, not the MCP stdio wire protocol, so the simplest way to expose them to an MCP-capable agent is through a generic HTTP/fetch MCP server. The reference fetch server works well for the read-only GET tools; point the agent at /api/mcp/tools so it can discover the rest. For the POST batch endpoints, call them directly with curl or wrap the API with an OpenAPI→MCP bridge.

Claude (Claude Code CLI)

Register a fetch MCP server, then ask Claude to use it against the manifest:

# add a generic fetch MCP server
claude mcp add fetch -- uvx mcp-server-fetch

# then, in a Claude Code session:
#   "Using the fetch tool, GET https://sec.midnightbsd.org/api/mcp/tools to
#    discover the security-advisory tools, then check whether openssl 3.0.2 is vulnerable."

Equivalent project config in .mcp.json (committed to your repo so teammates share it): 

{
  "mcpServers": {
    "fetch": { "command": "uvx", "args": ["mcp-server-fetch"] }
  }
}

Codex (OpenAI Codex CLI)

Add the server under [mcp_servers] in ~/.codex/config.toml:

[mcp_servers.fetch]
command = "uvx"
args = ["mcp-server-fetch"]

# then prompt Codex, e.g.:
#   "Fetch https://sec.midnightbsd.org/api/mcp/product/sendmail/version/5.58
#    and tell me if it is vulnerable."

Antigravity CLI

Antigravity reads MCP servers from its config (the same mcpServers JSON shape used by most MCP clients). Add: 

{
  "mcpServers": {
    "fetch": {
      "command": "uvx",
      "args": ["mcp-server-fetch"]
    }
  }
}

Then ask the agent to GET https://sec.midnightbsd.org/api/mcp/tools and call the security-advisory tools by URL. For automated package scanning, pipe your installed package list into POST /api/mcp/check (or the streaming variant) as shown in the examples above.

Tip: uvx ships with uv. Any MCP client with an HTTP/fetch tool can consume this API; the fetch server above is just the most portable option.
